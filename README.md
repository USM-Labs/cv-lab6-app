## Краткая документация

- `nginx/default.conf` - конфигурация веб-сервера nginx.
- `mounts/site/index.php` - главная страница PHP-приложения.
- `mounts/site/styles.css` - стили страницы.
- `.gitignore` - правило для каталога `mounts/site` с исключениями для демонстрационных файлов.

## Примеры использования

Создание сети:

```text
docker network create internal
```

Запуск backend:

```text
docker run -d --name backend --network internal -v "$PWD/mounts/site:/var/www/html" php:7.4-fpm
```

Запуск frontend:

```text
docker run -d --name frontend --network internal -p 80:80 -v "$PWD/mounts/site:/var/www/html" -v "$PWD/nginx/default.conf:/etc/nginx/conf.d/default.conf" nginx:1.23-alpine
```

Пример результата:

```text
HTTP/1.1 200 OK
Server: nginx/1.23.4
Content-Type: text/html; charset=UTF-8
```

## Безопасность

Пользовательский ввод на странице выводится только через `htmlspecialchars`, что защищает от XSS при отображении динамических данных. Контейнер backend не публикует порт на хост, поэтому доступен только внутри сети `internal`.

## Ответы на контрольные вопросы

1. Каким образом контейнеры могут взаимодействовать друг с другом?

   Контейнеры взаимодействуют через пользовательскую сеть Docker `internal`. nginx обращается к PHP-FPM по имени контейнера `backend` и порту `9000`.

2. Как контейнеры видят друг друга в рамках сети `internal`?

   Docker создает внутренний DNS. Поэтому контейнер `frontend` может использовать имя `backend` как сетевой адрес.

3. Почему необходимо было переопределять конфигурацию nginx?

   Стандартная конфигурация nginx не знает, куда отправлять PHP-файлы. В `default.conf` добавлен блок `location ~ \.php$`, который передает запросы в `backend:9000`.
